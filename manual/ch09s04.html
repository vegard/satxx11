Implementing the decision heuristic proper

With the code from the previous sections already in place, the last missing piece is the function that gets called by the solver when it is about to make a new decision. For this purpose, we overload operator() for our class and define it in terms of our two members random and vsids (which also overload operator()). It is really quite straightforward:

template<class Solver>
unsigned int operator()(Solver &s)
{
    /* Make a random decision 10% of the time */
    if (s.random() % 100 < 10)
        return random();

    return vsids();
}

If you paid attention, you might have noticed the call to s.random(). Here we are in fact making a reference to another designated plugin — the solver thread's random number generator. The solver parameter that gets passed to our operator() is guaranteed to have a member random that also supports operator().

With this, our plugin is complete.
